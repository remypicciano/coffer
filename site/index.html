The operating model

Small enough to explain.

Version 1 deliberately does less. Its security boundary is visible, documented, and testable.

  1. 01

    Choose one file

    Coffer reads the file locally and creates a fresh 256-bit key from the operating system’s secure random source.

  2. 02

    Review the destinations

    You choose where the protected container and its matching key will be written. Existing files are never silently replaced.

  3. 03

    Separate the pair

    Keep the .coffer container and .cofferkey in different places or channels. Both are required to restore.

Security boundary

Clear guarantees.
Clear limits.

Coffer authenticates before it writes plaintext, uses strict versioned parsers, and refuses overwrite races. It cannot make a compromised computer trustworthy.

Read the security policy →
Protects

Contents and filename metadata

The original filename, exact size, and file bytes are inside the authenticated encrypted payload.

Protects

Against accidental replacement

Temporary outputs are committed with no-clobber filesystem operations only after complete writes.

Does not protect

An already compromised session

Malware, administrators, keyloggers, screen capture, or process-memory inspection remain outside the boundary.

Cannot recover

A lost unlock key

There is no escrow, master key, account recovery, or developer backdoor. Back up the key deliberately.

Built to outlive the app

A documented recovery path.

The v1 byte layout is frozen and published with a deterministic compatibility vector. A standalone Python recovery utility provides an independent implementation.

What follows v1

Version 2 begins with the specification.

Key carriers and metadata sanitization change the security model. They are being designed as an explicit new container version—not slipped into v1 as ambiguous flags.

Read the v2 draft Threat model
Research

Ordinary-file key carriers

Derive wrapping material from exact carrier bytes while leaving the carrier untouched and useful in its normal application.

Stronger profile

Carrier + passphrase

Combine the carrier digest with Argon2id so possession of a public carrier is not sufficient by itself.

Privacy tooling

Verified sanitization

Create separate sanitized copies for formats where metadata removal can be performed and checked reliably.
